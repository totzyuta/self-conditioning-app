# Self Conditioning App

個人向けのコンディション記録とトレーニング記録を扱う Web アプリです。Vite + React で動き、ブラウザと Supabase（API 経由）で同期します。

---

## プロダクトとしての設計思想

### 観察優先

スコアやメモは「良い／悪い」を判定するためではなく、**日々の状態を残す・振り返る**ためのものとして扱います。数字は単位付きのメモに近く、絶対的な正解はアプリ側に置きません。

### コンディションスコアは「高いほど良い」ではない

0〜10 のコンディションは **ニュートラル 5.0 を軸**にした自己申告です。**高スコアを目指すゲーム**や **低スコアを問題視する評価**を前提にした UI 文言・配色にはしません。

- 帯ラベル（例: `Very low` / `Neutral` / `Very high`）は **程度の区分**であり、優劣を表しません。
- 色も **「良い＝緑・悪い＝赤」**のような意味付けは避け、後述のとおり **中立からの偏り**としてクール／ウォームを分けています。

### コンディションとトレーニングの分離

- **コンディション**（日次のスコア・専用メモ）は `conditions` 行として正規化。
- **トレーニング**（種目・トレーニング用メモ）は `training_sessions` / `training_items` 側。

どちらか一方のメモが他方を上書きしないよう、データと API ペイロードを分けています。

## デザイン思想（UI・色）

### トーン

全体は **ベージュ／オフホワイトの背景**（`--bg`）、**森緑**（`--green`）と **テラコッタ**（`--terra`）をアクセントにした落ち着いたトーンです。詳細は `src/styles/tokens.css` を参照してください。

### コンディションの連続色 `condColor`（数値・スライダー・オーブ・チャート線など）

- **5.0 付近 = 中立色**（`--muted` に相当するグレー寄り）。
- **5 より低い**ほど **クール色**（スレート系）へ連続変化。
- **5 より高い**ほど **ウォーム色**（レンガ系）へ連続変化。

いずれも **「中立からどれだけ離れているか」**の見た目であり、高い方が「成功色」にならないようにしています。旧来の「高スコア＝緑・低スコア＝オレンジ」のような **優劣を連想しやすいグラデーションは採用していません**。

実装: `src/lib/format.js` の `condColor`。

### 帯ラベル専用の離散色 `condLabelColor`

英語帯（`Very low` など）の **文字色だけ**に使う、彩度を抑えた **中間トーンのパレット**です（`tokens.css` の `--cond-band-*` と対応）。連続色の端点とトーンを揃えつつ、帯名の判読用に使い分けています。

実装: `src/lib/format.js` の `condLabelColor`。

### チャートまわり

- 折れ線の色は **期間内の平均スコア**に応じた `condColor` を使用し、統計数値の色は **本文色**（`--text`）を優先し、数値に「良さ」を色で載せないようにしています。

## 開発

```bash
npm install
npm run dev
```

本番ビルドは `npm run build`。Vercel 上では `api/v2/state` と Supabase、環境変数（`SUPABASE_URL`、`SYNC_PASSWORD` など）の設定が必要です。詳細は [ARCHITECTURE.md](ARCHITECTURE.md) を参照してください。

# ond（オンド）

## Concept

名前の意味
「Onde（フランス語：波）」＋「温度（日本語：状態）」

心のゆらぎを一つの「波」として捉え、その時々の自分の「温度感」をそのまま記録する。
良い・悪いの評価を捨て、ただ今の自分を観測するためのセルフコンディショニング・ツールです。

ステートメント：自分を「観測」する、ニュートラルな居場所。
私たちは日々、絶え間なく変化する波の中にいます。
やる気に満ちた熱い日もあれば、静かに沈み込むような冷たい日もある。
しかし、その波に「正解」はありません。

ond は、スコアを高めるためのゲームではありません。
中央にある 5.0（Neutral） を軸に、自分という現象が今、どちらに振れているかを静かに眺めるための場所です。

心と体の、心地よい分離
「こころ」のコンディションと「からだ」のトレーニング。
これらを混同せず、それぞれの文脈で記録することで、自分をより多層的に理解できます。

色彩が語る、ありのままの状態
「良い＝緑」「悪い＝赤」という従来の記号を排除しました。
森緑（Forest Green）とテラコッタ（Terracotta）を基調としたアースカラーのグラデーションは、中立から離れた「偏り」を視覚化するだけ。
どんな数値であっても、それはあなたを構成する大切な記録の一部です。

積み重なる、日々の地層
感情や体調の波を、ただ線で結んでいく。
数ヶ月後、そこにはあなただけの「バイオリズム」という地図が描かれています。
自分をコントロールするのではなく、自分を理解し、共に歩んでいくために。

## UI / UX Philosophy
- Neutrality: 5.0 を起点とした、評価しない UI。
- Observation First: 改善よりも、まずは「観察」に重きを置く。
- Organic Tones: 身体性に訴えかける、落ち着いた配色と手触り感。

## プロダクトとしての設計思想

### 観察優先

スコアやメモは「良い／悪い」を判定するためではなく、**日々の状態を残す・振り返る**ためのものとして扱います。数字は単位付きのメモに近く、絶対的な正解はアプリ側に置きません。

### コンディションスコアは「高いほど良い」ではない

0〜10 のコンディションは **ニュートラル 5.0 を軸**にした自己申告です。**高スコアを目指すゲーム**や **低スコアを問題視する評価**を前提にした UI 文言・配色にはしません。

帯ラベル（例: Very low / Neutral / Very high）は **程度の区分**であり、優劣を表しません。色も **「良い＝緑・悪い＝赤」**のような意味付けは避け、**中立からの偏り**としてクール寄り／ウォーム寄りを分けています。

### コンディションとトレーニングの分離

コンディション用の記録と、トレーニング用の記録は **別の文脈**として扱います。どちらか一方のメモが他方をすり替えないよう、データ上も切り分けています。

## デザイン思想（UI・色）

### トーン

全体は **ベージュ／オフホワイトの背景**に、**森緑**と **テラコッタ**をアクセントとした、落ち着いたトーンです。

### 数値まわりの連続色

**5.0 付近は中立のグレー寄り**とし、**5 より低いほどクール**（スレート系）、**5 より高いほどウォーム**（レンガ系）へ、連続的に変化させます。いずれも **中立からどれだけ離れているか**の見た目であり、高い方が「成功色」にならないようにしています。旧来の「高スコア＝緑・低スコア＝オレンジ」のような、優劣を連想しやすいグラデーションにはしていません。

### 帯ラベル（英語の程度表示）

帯名の文字だけ、彩度を抑えた **中間トーンのパレット**で区別します。連続色の雰囲気と揃えつつ、読みやすさ用の使い分けです。

### 推移グラフ

折れ線は **キーカラーの緑で統一**し、スコアの高低で色が変わらないようにしています。推移そのものは「同じ軸で眺める」ための見立てです。統計欄の数値は **本文色**に寄せ、数値に「良さ」を色で載せないようにしています。
